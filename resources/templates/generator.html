Generator API



When the URI /generator.json is accessed with an AJAX request, the server generates a graph with satellites in different locations. Depending on the request, the locations can be pseudo-randomly selected and at different altitudes or at fixed altitudes forming certain geometric figures. The graph returned by the server is a hashmap with the keys nodes and edges, which is returned as serialized JSON. The content of nodes and edges is explained in more detail below.


Data set types


Random data

Currently 4 different parameters can be given with the request to be passed to the generator to create random values:


satelliteCount
Amount of satellite nodes to be generated. Must be integer and positive. Defaults to 5.
minAltitude
Minimum altitude above the planet surface for a satellite. Must be a positive number. Defaults to 300.0.
maxAltitude
Maximum altitude above the planet surface for a satellite. Must be a positive number and larger than minAltitude. Defaults to 700.0.
planetRadius
Radius of the planet (sphere) above which satellites are generated. Must be a positive number. Defaults to 6371.0 (Earth's mean radius in kilometers).

Polyhedrons

It is also possible to generate satellite constellations in form of platonic solids, where a satellite will be placed at each vertex of the chosen polyhedron. 3 different parameters can be given for generating polyhedrons:

polyhedron
Name of the polyhedron. Choices are:
  • tetrahedron
  • cube
  • octahedron
  • dodecahedron
  • icosahedron
The name must be exact, otherwise the server will generate random data as described above.
altitude
Altitude above the planet surface for a satellite. Must be a positive number. Defaults to 0.0.
planetRadius
Radius of the planet (sphere) above which satellites are generated. Must be a positive number. Defaults to 6371.0 (Earth's mean radius in kilometers).

Incorrect input

The server will replace all incorrect parameter values with their corresponding default values. Also, if the maximum altitude is less than the minimum altitude, the default values for both altitudes will be used.

Data set representation


Nodes

Nodes are given as a sequence of hashmaps, where one hashmap corresponds to a node with the keys name and pos. The position is a three dimensional vector in cartesian coordinates following the ECEF system. Note that in this app the Earth is modeled as a perfect sphere and not an ellipsoid, which would be more realistic. Below is the first node in raw format and a table of all nodes.

Edges

Edges are given as a sequence (vector) of vectors, where one vector correspond to an edge. In this vector, indexes 0 and 1 contain the nodes which are connected and index 2 contains the float value corresponding to the distance between these nodes measured in kilometers. Index 3 contains a hashmap with only one key is_solution_path, which maps to true if the corresponding edge is part of the shortest path between nodes named "START" and "END", else false.
